# 路线图

蒙氏空间的路线图遵循三个原则：demo 先行、隐私先行、观察先行。

当前首发版只承诺 Phase 1 Dummy Pipeline。后续阶段是方向，不是当前发布承诺。

## Phase 1: Dummy Pipeline

目标：在不依赖真实儿童数据的情况下，跑通本地行为数据管线。

计划交付：

- 本地视频文件或纯本地输入适配。
- 抽帧流程。
- Mock MontessoriBehaviorLog JSON。
- 本地存储或可替换存储说明。
- Markdown/HTML 合成日报。
- 基础隐私说明：数据目录、删除方式、不会上传什么。

成功标准：

- 完成“输入 -> 抽帧 -> JSON -> 存储 -> 报告”的本地闭环。
- 不使用真实儿童数据也能演示项目价值。
- 开发者能理解并复现最小管线。

## Phase 2: 本地视觉模型与空间处理

方向：将 mock 推理逐步替换为本地 VLM、图像裁剪和空间处理流程。

边界：仍然使用合成样例或非敏感公开素材，不把真实儿童数据作为默认评测资产。

## Phase 3: 蒙氏语义与家庭反馈

方向：打磨观察语言、家长纠错、日报模板和低频提醒策略。

边界：保持观察型输出，不进入儿童评分、发展诊断、情绪识别或自动惩罚。

## Phase 4: 低侵入硬件反馈

方向：在数据管线稳定后，探索 LED、极简 TTS 或本地音频等轻量反馈。

边界：硬件反馈可关闭、可解释，不作为 v1 阻塞项，也不服务持续控制感。

## 明确不在当前路线图中

- 开放真实儿童原始数据集。
- 默认云端集中训练。
- 儿童评分、心理诊断、情绪识别或能力排名。
- 自动惩罚、训斥或强干预。
- 完整商业硬件套装。
