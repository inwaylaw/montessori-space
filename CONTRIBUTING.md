# 贡献指南

感谢你关注蒙氏空间。这个项目当前处在概念倡议和 Phase 1 Dummy Pipeline 之前的先行发布阶段，因此贡献重点不是扩大功能，而是把边界、文档、合成样例和本地可复现路径做扎实。

## 可以贡献的内容

- 文档改进：README、路线图、隐私边界、贡献边界、术语表述。
- 隐私边界审阅：指出可能造成监控化、评分化、诊断化或数据外流误解的文字。
- Schema 建议：围绕低风险观察字段改进 MontessoriBehaviorLog。
- 合成样例：构造不含真实儿童数据的行为日志、日报和测试场景。
- 设备本地路径反馈：说明某类设备是否支持 UVC、RTSP、本地文件流或其他纯本地输入。
- Prompt 语言建议：让输出保持“观察、需确认、可纠错”，避免评分、诊断和命令式干预。

## 暂不接受的内容

- 真实儿童原始视频、音频、照片或截图。
- 完整家庭行为日志、家庭空间布局、住址、聊天记录或可识别家庭成员的信息。
- 基于真实儿童数据的评测样例。
- 儿童评分、发展诊断、心理画像、情绪识别或惩罚触发逻辑。
- 默认云端上传、云端集中训练或第三方持续接管原始数据的方案。

## 数据提交规则

真实儿童数据不进仓库。这个规则适用于 issue、PR、commit、附件、截图、终端日志、演示文件和示例数据。

如果需要说明问题，请使用以下方式替代：

- 合成 JSON。
- 手写伪数据。
- 不含人物、住址和家庭识别信息的抽象场景描述。
- 公开可用且不涉及真实儿童隐私的素材。

## 贡献前自检

提交前请确认：

- 没有上传真实儿童、照料者或家庭可识别数据。
- 没有把 AI 输出写成儿童评分、诊断或绝对事实。
- 没有把 v1 描述成完整产品、硬件套装或云端平台。
- 没有暗示项目当前开放或收集真实儿童数据集。
- 示例文件明确标注为合成内容。
